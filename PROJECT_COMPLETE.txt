🎯 CORE COMPONENTS
═══════════════════════════════════════════════════════════════════════════════

MAIN ENGINE
├─ HyperDict (core/engine.ts)
│  └─ registerDictionary() → init() → lookup() → getDefinition()
│
├─ Dictionary Manager (core/dictionary.ts)
│  └─ Lazy loading, multi-dict support, worker coordination
│
└─ Web Worker (core/worker.ts)
   └─ Off-thread computation, cache management, decompression


ALGORITHMS (Ultra-Optimized)
├─ ShaekeebBinarySearch
│  └─ Direct byte comparison, no temp strings, UTF-8 safe
│
├─ ShaekeebPrefixIndex  
│  └─ First 2-byte UTF-8 prefix mapping, 1000x scope reduction
│
├─ ShaekeebBloomFilter
│  └─ Probabilistic negative lookup, configurable FP rate
│
└─ ShaekeebLRUCache
   └─ Decompressed block caching, configurable size


FILE SUPPORT
├─ IDX Parser (ShaekeebIdxParser)
│  └─ StarDict .idx format, TypedArray output
│
├─ IFO Parser (ShaekeebIfoParser)
│  └─ Dictionary metadata extraction, validation
│
├─ DictZip Header Parser (ShaekeebDictZipHeaderParser)
│  └─ Gzip random access, block offset extraction
│
├─ Block Reader (ShaekeebBlockReader)
│  └─ Individual block fetching, decompression
│
└─ Range Fetcher (ShaekeebRangeFetcher)
   └─ HTTP Range requests, partial downloads


DATA STRUCTURES
├─ ShaekeebTypedIndex
│  └─ Binary searchable index with SharedArrayBuffer support
│
├─ TypedIndexBuilder
│  └─ Builder pattern for efficient index creation
│
└─ TypedIndexReader
   └─ Safe index reading without allocations
